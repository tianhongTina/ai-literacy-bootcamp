## 📖 课程内容

### 1. AI 写作的正确姿势

**三步法**：

```
第一步：明确目标 → 写给谁看？达成什么目的？
第二步：提供素材 → 给 AI 足够的上下文
第三步：迭代优化 → 不满意就追问、调整
```

### 2. 常见文档类型

#### 类型 1：工作邮件

**场景**：向客户发送项目进度更新

```
请帮我写一封项目进度邮件：

收件人：客户方项目经理
背景：我们的项目进入第二阶段
要传达的信息：
1. 第一阶段已完成，交付物已发送
2. 第二阶段将于下周启动
3. 需要客户提供 XX 资料

语气：专业、友好
```

#### 类型 2：会议纪要

**场景**：整理会议录音/笔记

```
这是一场产品评审会的会议记录：
[粘贴会议笔记或录音转文字]

请整理成规范的会议纪要，包含：
1. 会议基本信息（时间、参与人、主题）
2. 讨论要点（分条列出）
3. 决策事项（明确标注）
4. 待办事项（责任人 + 截止时间）
```

#### 类型 3：产品文档

**场景**：PRD（产品需求文档）

```
我需要写一份 PRD，功能是 [功能描述]

背景：
- 用户痛点：[具体痛点]
- 业务目标：[目标]
- 使用场景：[场景]

请按以下结构输出：
1. 需求背景
2. 用户故事
3. 功能描述
4. 交互流程
5. 验收标准
```

### 3. 风格迁移技巧

**让 AI 模仿你的风格**：

```
这是我之前写的 3 篇文章：
[文章1]
[文章2]
[文章3]

请分析我的写作风格特点，然后用相同风格写一篇关于 [主题] 的文章。
```

### 4. 内容改写

**场景**：同一内容，多种表达

```
请将以下内容改写成 3 个版本：
[原始内容]

版本1：正式、专业（用于对外报告）
版本2：轻松、口语化（用于内部分享）
版本3：精简、要点式（用于 PPT）
```

## 💡 岗位专属案例

### 产品经理

**需求文档快速生成**

```
我需要为 [功能名称] 写需求文档

用户调研发现：
- [发现1]
- [发现2]

竞品分析：
- [竞品做法]

请帮我生成：
1. 需求背景（为什么做）
2. 目标用户（给谁用）
3. 核心功能（做什么）
4. 优先级排序（先做什么）
```

### 运营

**活动方案撰写**

```
我要策划一个 [活动类型] 活动

目标：[具体目标，如 "新增 1000 个注册用户"]
预算：[预算范围]
时间：[活动周期]

请帮我生成活动方案，包含：
1. 活动主题和 slogan
2. 活动机制设计
3. 推广渠道和排期
4. 效果预估
```

### HR

**招聘 JD 撰写**

```
我需要招聘一名 [岗位名称]

岗位要求：
- 工作年限：[X年]
- 核心技能：[技能列表]
- 工作内容：[主要职责]

公司信息：
- 行业：[行业]
- 规模：[人数]
- 福利：[福利亮点]

请生成一份吸引人的招聘 JD，突出 [公司优势]
```

## 🎯 实战练习

### 练习 1：写作流程实操

选择一个你需要写的文档，用 AI 辅助完成，记录：
1. 你的初始提示词

3. 你的修改意见
4. 最终版本

用 AI 生成 10 条社交媒体文案，要求：

- 不同角度
- 不同长度（短/中/长）

## ⚠️ 质量把控要点

### 检查点 1：事实准确性

- ❌ AI 可能编造数据、引用不存在的来源
- ✅ 所有数据、案例、引用都要人工核实

### 检查点 2：逻辑连贯性

- ❌ AI 可能前后矛盾、论证不严密
- ✅ 检查论点是否支撑结论、逻辑链是否完整

### 检查点 3：风格一致性

- ❌ AI 可能在同一文档中风格跳跃
- ✅ 确保语气、用词、格式统一

## ❓ 常见问题

**Q: AI 生成的内容可以直接用吗？**

A: 不建议。AI 是"助手"不是"替代"。生成的内容需要人工审核、修改、补充。

**Q: 如何避免 AI 生成的内容千篇一律？**

A: 提供具体的上下文、案例、数据，让 AI 基于真实信息创作。

**Q: AI 写作会被检测出来吗？**

A: 经过人工修改和补充的内容，通常不会被检测为 AI 生成。关键是加入你的思考和独特见解。
